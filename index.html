Capabilities

Everything you need,
nothing you don't.

⊞
Full CRUD on Item Data

Add, edit, view, duplicate, and delete regulatory records. Composite primary key on Item ID + Country + Material + Regulation ensures no duplicate entries.

🌍
Lookup Table Management

Manage Countries, Material Groups, Declaration types, and Regulation frameworks independently. All linked by ID to item data records.

⚗️
Contaminant Limits

Track per-field limits for Aflatoxin B1, Ochratoxin A, PAH, PCBs, Heavy Metals, Mercury, Cadmium, Arsenic, Pesticides and more — each with a value and source link.

🔍
Search & Filter

Live search with debounce across item IDs, remarks, and websites. Dropdown filters for Country, Material, and Regulation type with a one-click clear.

🖨
Print & Export

Print-ready record view opens in a new window — clean layout with all 25 fields. One-click copy buttons for every value and link in the view modal.

📦
Fully Offline

SQLite database lives next to the .exe — no server, no internet, no accounts. Data is yours. Auto-seeds sample countries, materials, and regulations on first run.

Getting Started

Setup in four steps.

01
Install Prerequisites

You'll need Go 1.21+, Node.js 18+, the Wails CLI, and TDM-GCC (required for CGO / SQLite on Windows).

# Install Wails CLI
go install github.com/wailsapp/wails/v2/cmd/wails@latest

# Verify
wails doctor
02
Clone & Install Dependencies

Clone the repository, then install both Go modules and npm packages.

git clone https://github.com/nandhuzz/synthite-food-data
cd synthite-food-data

go mod tidy

cd frontend
npm install
cd ..
03
Run in Development Mode

Hot-reload dev server. Delete the wailsjs stub first so Wails regenerates the real bindings from your Go code.

# Delete the committed stub (Windows)
del frontend\wailsjs\go\main\App.js

wails dev
04
Build the Windows Executable

Produces a standalone .exe. The SQLite database is auto-created next to the binary on first launch and seeded with sample data.

wails build
# Output: build/bin/synthite-food-data.exe
